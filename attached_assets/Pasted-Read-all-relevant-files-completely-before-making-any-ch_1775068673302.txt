FIX 1 — Runtime crash: "job.salary.match is not a function"

Find every place in the codebase where .match(), 
.replace(), .includes(), .toLowerCase() or any other 
string method is called directly on job.salary or any 
salary-related field without first verifying it is a string.

Fix each one with this safety pattern:
const salaryStr = typeof job.salary === 'string'
  ? job.salary
  : job.salary != null
    ? String(job.salary)
    : ''

Then use salaryStr for all string operations instead of 
calling methods directly on job.salary.

Apply this same pattern to any other field that could 
be non-string but has string methods called on it 
directly — job.compensation, job.salary_min, 
job.salary_max, etc.

FIX 2 — Glassdoor and ZipRecruiter TLSClientException

Do not change any proxy format code. Instead:

Add a pre-flight test at the start of the JobSpy run 
that makes one small test request to Glassdoor and one 
to ZipRecruiter before running the full search matrix.

If the test request succeeds → run that source normally.
If the test request fails with TLSClientException or 
any connection error → log clearly:
"Glassdoor: connection failed — skipping this run"
"ZipRecruiter: connection failed — skipping this run"
And skip that source entirely for this run rather than 
attempting 15 searches that will all fail.

This way working sources always run, broken sources 
are skipped gracefully, and we never waste time on 
guaranteed failures.
