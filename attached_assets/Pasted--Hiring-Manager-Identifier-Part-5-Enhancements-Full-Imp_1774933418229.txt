## ENHANCEMENT 1: Apollo.io API Integration (Secondary Data Source)

### What It Does
When Gemini search grounding returns weak results (few or no LinkedIn profiles), automatically fall back to Apollo.io's People Search API to find the hiring manager. Also use Apollo to enrich existing hiring manager records with additional data (email, phone, company details).

### Setup

`APOLLO_API_KEY` is already in your Replit Secrets.

### Modify the Existing `/api/hiring-manager/identify` Flow

Update the existing identification endpoint to include Apollo as a fallback. The new flow:

```
Step 1: Claude analyzes JD (unchanged)
   ↓
Step 2: Gemini search with grounding (unchanged)
   ↓
Step 2.5 (NEW): Check Gemini result quality
   - If Gemini returned 3+ LinkedIn profiles with names and titles → skip Apollo, proceed to Step 3
   - If Gemini returned 0-2 usable results → call Apollo People Search with the same title keywords
   - Merge Apollo results with Gemini results into one combined results block
   ↓
Step 3: Claude identifies hiring manager from combined results (unchanged, just more data)
   ↓
Step 3.5 (NEW): Apollo enrichment
   - Once Claude picks the hiring manager, call Apollo Enrich with their name + company domain
   - If Apollo returns an email, phone, or additional data, save it to the hiring_managers record
   ↓
Step 4: Save to database (add Apollo fields)
```

The key logic for Step 2.5:

```typescript
// After Gemini search results come back, count usable LinkedIn profiles
```

### Helper Functions for API Usage Tracking

```typescript
const trackUsage = async (service: string, endpoint: string, credits: number) => {
  await db.query(
    'INSERT INTO api_usage (service, endpoint, credits_used) VALUES ($1, $2, $3)',
    [service, endpoint, credits]
  );
};

const getMonthlyUsage = async (service: string) => {
  const result = await db.query(
    `SELECT COALESCE(SUM(credits_used), 0) as total 
     FROM api_usage 
     WHERE service = $1 
     AND created_at >= date_trunc('month', NOW())`,
    [service]
  );
  return parseInt(result.rows[0].total);
};

const getDailyUsage = async (service: string) => {
  const result = await db.query(
    `SELECT COALESCE(SUM(credits_used), 0) as total 
     FROM api_usage 
     WHERE service = $1 
     AND created_at >= date_trunc('day', NOW())`,
    [service]
  );
  return parseInt(result.rows[0].total);
};
```

## ENHANCEMENT 2: Email Finder (Hunter.io Integration)

### What It Does
Once you have a hiring manager's name and company, use Hunter.io to find their email address. Hunter is better than Apollo for email specifically because it verifies the email pattern against the company's domain.

### Setup

`HUNTER_API_KEY` is already in your Replit Secrets.

### New API Endpoint: `/api/hunter/find-email`

**Method**: POST

```typescript
const findEmail = async (firstName: string, lastName: string, companyDomain: string) => {
  // Step 1: Find the email
  const finderResponse = await fetch(
    `https://api.hunter.io/v2/email-finder?domain=${companyDomain}&first_name=${firstName}&last_name=${lastName}&api_key=${process.env.HUNTER_API_KEY}`
  );
  const finderData = await finderResponse.json();
  
  if (!finderData.data?.email) {
    return { email: null, confidence: 0, verified: false };
  }
  
  const email = finderData.data.email;
  const confidence = finderData.data.confidence;  // 0-100
  
  // Step 2: Verify the email is deliverable
  const verifyResponse = await fetch(
    `https://api.hunter.io/v2/email-verifier?email=${email}&api_key=${process.env.HUNTER_API_KEY}`
  );
  const verifyData = await verifyResponse.json();
  
  // Track usage
  await trackUsage('hunter', 'email_finder', 1);
  await trackUsage('hunter', 'email_verifier', 1);
  
  return {
    email: email,
    confidence: confidence,
    verified: verifyData.data?.status === 'valid',
    status: verifyData.data?.status  // 'valid', 'invalid', 'accept_all', 'webmail', 'disposable', 'unknown'
  };
};
```

## ENHANCEMENT 3: LinkedIn Message Drafts

### What It Does
Auto-generate a personalized LinkedIn connection request message for the identified hiring manager. Uses the same cold-email philosophy as the cover letter — short, specific, shows you understand their business, asks for a low-commitment next step.

### Important Constraint
LinkedIn connection request messages have a **300 character limit**. This is NOT a cover letter. It's a micro-pitch. Every word counts.

### Database Changes

```sql
-- Store generated LinkedIn messages
CREATE TABLE linkedin_messages (
  id SERIAL PRIMARY KEY,
  hiring_manager_id INTEGER REFERENCES hiring_managers(id),
  job_id INTEGER,                     -- reference to the saved job
  message_text TEXT NOT NULL,
  char_count INTEGER NOT NULL,        -- must be <= 300
  version INTEGER DEFAULT 1,          -- allow multiple drafts
  status TEXT DEFAULT 'draft',        -- 'draft', 'sent', 'accepted', 'no_response'
  sent_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW()
);
```

### System Prompt (Store in `system_prompts` Table)

**prompt_name**: `linkedin_connection_message`

**prompt_text**:

```
You write LinkedIn connection request messages for job applicants reaching out to hiring managers. These messages must be under 300 characters — LinkedIn's hard limit.

## THE RULES

1. MUST be 300 characters or fewer. Count carefully. If it's 301, it won't send.
2. No greeting ("Hi [Name]" or "Dear [Name]") — LinkedIn already shows who you are
3. No "I'd love to connect" — that's what the button does
4. No "I came across your profile" — everyone says this
5. No "I'm passionate about" — nobody cares
6. One specific thing about THEIR company or team. Not generic.
7. One line about what YOU bring. Metric or proof point.
8. One ask or statement of intent. "Exploring the [role title] opening" or "Would value a quick conversation about [specific thing]."
9. Sound like a text message from a professional, not a cover letter
10. Use contractions. Write like you talk.

## WHAT YOU RECEIVE

- Hiring manager's name and title
- Company name
- Role being applied for
- Company research notes
- Applicant's key proof points (from tailored resume)

## STRUCTURE (3 sentences, that's it)

Sentence 1: Something specific about their company/team/market that shows you did homework (NOT "I see you're hiring for X")
Sentence 2: Your most relevant proof point — one number, one company
Sentence 3: What you want — "Exploring the [role]" or "Would love 15 min to discuss"

## EXAMPLES OF GOOD MESSAGES

"Akamai's push into new verticals for cloud security is smart timing. Built $5M in net-new pipeline doing exactly that at ClearDATA, co-selling with AWS. Exploring the Southeast SAE role — happy to share perspective."

"Noticed CoreWeave is scaling the sales org fast. Closed 34 net-new logos at Carbon Black at 137% quota — strong hunter background. Would value a quick chat about the enterprise AE opening."

## EXAMPLES OF BAD MESSAGES

"Hi Jane, I'm a passionate sales professional with 6+ years of experience in enterprise SaaS sales. I noticed Akamai is hiring and I believe my background aligns well with your team's needs. I'd love to connect and learn more about the opportunity."
(Too long, generic, starts with "Hi", "passionate", "I believe", "I'd love to connect" — every red flag)

## OUTPUT

Return ONLY valid JSON:

{
  "message": "The connection request text",
  "char_count": 187,
  "alternative": "A second version with a slightly different angle",
  "alt_char_count": 203
}

Both versions MUST be under 300 characters. Count carefully — include spaces and punctuation in the count.
```

### New API Endpoint: `/api/linkedin-message/generate`

**Method**: POST

```typescript
const generateLinkedInMessage = async (
  hiringManagerId: number,
  jobId: number
) => {
  // Fetch hiring manager, job details, and company research from database
  const hiringManager = await db.query('SELECT * FROM hiring_managers WHERE id = $1', [hiringManagerId]);
  const job = await db.query('SELECT * FROM saved_jobs WHERE id = $1', [jobId]);  // adjust table name
  const application = await db.query('SELECT * FROM job_applications WHERE job_id = $1', [jobId]);  // if exists
  
  // Get the system prompt from database
  const systemPrompt = await db.query(
    "SELECT prompt_text FROM system_prompts WHERE prompt_name = 'linkedin_connection_message'"
  );
  
  const response = await fetch("https://api.anthropic.com/v1/messages", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "x-api-key": process.env.ANTHROPIC_API_KEY,
      "anthropic-version": "2023-06-01"
    },
    body: JSON.stringify({
      model: "claude-opus-4-6",
      max_tokens: 512,
      system: systemPrompt.rows[0].prompt_text,
      messages: [
        {
          role: "user",
          content: `HIRING MANAGER:
Name: ${hiringManager.rows[0].full_name}
Title: ${hiringManager.rows[0].title}
Company: ${hiringManager.rows[0].company}

ROLE I'M APPLYING FOR:
${job.rows[0].role_title}

COMPANY RESEARCH:
${application?.rows[0]?.company_research || 'No research available — use what you know about the company'}

MY KEY PROOF POINTS (pick the single most relevant one):
${application?.rows[0]?.tailored_resume || job.rows[0].job_description}

Generate the LinkedIn connection request message.`
        }
      ]
    })
  });
  
  const data = await response.json();
  const text = data.content.map(c => c.text || '').join('');
  const cleaned = text.replace(/```json|```/g, '').trim();
  const parsed = JSON.parse(cleaned);
  
  // Validate character count
  if (parsed.message.length > 300) {
    console.warn('Message exceeded 300 chars, requesting rewrite');
  }
  
  // Save both versions to database
  await db.query(
    'INSERT INTO linkedin_messages (hiring_manager_id, job_id, message_text, char_count, version) VALUES ($1, $2, $3, $4, 1)',
    [hiringManagerId, jobId, parsed.message, parsed.message.length]
  );
  
  if (parsed.alternative) {
    await db.query(
      'INSERT INTO linkedin_messages (hiring_manager_id, job_id, message_text, char_count, version) VALUES ($1, $2, $3, $4, 2)',
      [hiringManagerId, jobId, parsed.alternative, parsed.alternative.length]
    );
  }
  
  await trackUsage('anthropic', 'linkedin_message', 1);
  
  return parsed;
};
```

### New API Endpoint: `/api/accuracy/stats`

**Method**: GET

Returns accuracy statistics:

```typescript
const getAccuracyStats = async () => {
  const stats = await db.query(`
    SELECT 
      COUNT(*) as total_identifications,
      COUNT(*) FILTER (WHERE was_correct = true) as correct,
      COUNT(*) FILTER (WHERE was_correct = false) as incorrect,
      ROUND(
        COUNT(*) FILTER (WHERE was_correct = true)::decimal / 
        NULLIF(COUNT(*), 0) * 100, 1
      ) as accuracy_pct,
      
      -- Accuracy by confidence level
      COUNT(*) FILTER (WHERE original_confidence = 'high' AND was_correct = true) as high_conf_correct,
      COUNT(*) FILTER (WHERE original_confidence = 'high') as high_conf_total,
      COUNT(*) FILTER (WHERE original_confidence = 'medium' AND was_correct = true) as med_conf_correct,
      COUNT(*) FILTER (WHERE original_confidence = 'medium') as med_conf_total,
      COUNT(*) FILTER (WHERE original_confidence = 'low' AND was_correct = true) as low_conf_correct,
      COUNT(*) FILTER (WHERE original_confidence = 'low') as low_conf_total,
      
      -- Accuracy by source
      COUNT(*) FILTER (WHERE search_source = 'gemini' AND was_correct = true) as gemini_correct,
      COUNT(*) FILTER (WHERE search_source = 'gemini') as gemini_total,
      COUNT(*) FILTER (WHERE search_source LIKE '%apollo%' AND was_correct = true) as apollo_correct,
      COUNT(*) FILTER (WHERE search_source LIKE '%apollo%') as apollo_total
      
    FROM identification_feedback
  `);
  
  // Get top correction reasons separately
  const reasons = await db.query(`
    SELECT correction_reason, COUNT(*) as count
    FROM identification_feedback
    WHERE was_correct = false AND correction_reason IS NOT NULL
    GROUP BY correction_reason
    ORDER BY count DESC
    LIMIT 5
  `);
  
  return { stats: stats.rows[0], top_correction_reasons: reasons.rows };
};
```

## ENHANCEMENT 5: Company Org Chart Cache

### What It Does
If you apply to multiple roles at the same company (e.g., two different AE roles at Akamai), don't burn Gemini/Apollo searches re-identifying the same leadership team. Cache the leadership profiles by company and reuse them.

### Database Changes

```sql
-- Cache company leadership profiles
CREATE TABLE company_leadership_cache (
  id SERIAL PRIMARY KEY,
  company_name TEXT NOT NULL,
  company_domain TEXT,
  person_name TEXT NOT NULL,
  person_title TEXT NOT NULL,
  linkedin_url TEXT,
  department TEXT,                   -- 'Sales', 'Marketing', 'Engineering', etc.
  region TEXT,
  seniority_level TEXT,             -- 'C-Suite', 'VP', 'Director', 'Manager'
  source TEXT,                      -- 'gemini', 'apollo', 'manual'
  still_active BOOLEAN DEFAULT TRUE,  -- flag if someone left the company
  last_verified_at TIMESTAMP DEFAULT NOW(),
  expires_at TIMESTAMP DEFAULT (NOW() + INTERVAL '90 days'),  -- cache for 90 days
  created_at TIMESTAMP DEFAULT NOW(),
  CONSTRAINT unique_company_person UNIQUE (company_name, person_name)
);

-- Index for fast lookups
CREATE INDEX idx_company_cache_lookup 
  ON company_leadership_cache(company_name, department, seniority_level);
```

### Logic Changes

```typescript
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "x-api-key": process.env.ANTHROPIC_API_KEY,
      "anthropic-version": "2023-06-01"
    },
    body: JSON.stringify({
      model: "claude-opus-4-6",
      max_tokens: 2048,
      system: `Extract all leadership profiles (Directors, VPs, Heads, C-suite) from the search results. Return ONLY a JSON array:
[
  {
    "person_name": "First Last",
    "person_title": "Their title",
    "linkedin_url": "URL or null",
    "department": "Sales/Marketing/Engineering/Product/Finance/People/Unknown",
    "region": "region if mentioned or null",
    "seniority_level": "C-Suite/SVP/VP/Director"
  }
]
Only include people who appear to currently work at ${companyName} in leadership roles. No recruiters, no ICs. Return ONLY the JSON array.`,
      messages: [
        { role: "user", content: `Company: ${companyName}\n\nSearch Results:\n${searchResults}` }
      ]
    })
  });
  
  const data = await extractResponse.json();
  const text = data.content.map(c => c.text || '').join('');
  const cleaned = text.replace(/```json|```/g, '').trim();
  
  try {
    const leaders = JSON.parse(cleaned);
    
    for (const leader of leaders) {
      await db.query(`
        INSERT INTO company_leadership_cache 
        (company_name, person_name, person_title, linkedin_url, department, seniority_level, region, source, last_verified_at, expires_at)
        VALUES ($1, $2, $3, $4, $5, $6, $7, 'gemini', NOW(), NOW() + INTERVAL '90 days')
        ON CONFLICT (company_name, person_name) 
        DO UPDATE SET 
          person_title = EXCLUDED.person_title,
          linkedin_url = EXCLUDED.linkedin_url,
          department = EXCLUDED.department,
          seniority_level = EXCLUDED.seniority_level,
          region = EXCLUDED.region,
          last_verified_at = NOW(),
          expires_at = NOW() + INTERVAL '90 days'
      `, [companyName, leader.person_name, leader.person_title, leader.linkedin_url, 
          leader.department, leader.seniority_level, leader.region]);
    }
    
    console.log(`Cached ${leaders.length} leadership profiles for ${companyName}`);
  } catch (e) {
    console.error('Failed to parse/cache leadership profiles:', e);
  }
};
```
